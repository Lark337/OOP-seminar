Здесь нарушены принцип единственной ответственности и принцип инверсии зависимостей.
Для решения первой проблемы был создан дополнительный класс UserSave, который отвечает за сохранение и был сделан static, чтобы можно было вызывать его без создания экземпляра класса.
Для решения второй проблемы метод report стал соеденен c классом через интерфейс Reportable.
